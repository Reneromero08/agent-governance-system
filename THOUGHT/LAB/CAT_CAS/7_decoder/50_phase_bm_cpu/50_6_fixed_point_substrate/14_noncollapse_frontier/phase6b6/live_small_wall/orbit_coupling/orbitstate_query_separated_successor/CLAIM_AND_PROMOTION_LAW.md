Status: `FROZEN_DESIGN_CONSTRAINT`

```text

QUERY_SEPARATED_RELATIONAL_CARRIER_NOT_ESTABLISHED
QUERY_SEPARATED_RELATIONAL_CARRIER_CUSTODY_INVALID
```

```text

```

## Candidate Claim

`QUERY_SEPARATED_RELATIONAL_CARRIER_CANDIDATE` means:

```text
Under the frozen query-separated schedule, a relation-symmetric physical preparation
and receiver-only post-source query produced the predeclared relational contrast,
with every mandatory gate in the single authoritative gate registry passing at the
declared restoration tier.
```

This remains an intermediate claim. It is route/device/session/query-family scoped.
It means "not reconstructed by the frozen adversary suite", not "absolute scalar
impossibility". It is not a Wall crossing by itself.

```text

```

## Authoritative Classification Timing

Target-side results are provisional. Authoritative classification may occur only
after:

```text
target custody passes
copy-back verifies
executed source reconstructs
required evidence set verifies
remote cleanup or retention state verifies
post-run audit passes
```

If target emits a candidate result and controller custody fails, the result is
custody-invalid and no scientific class is retained.

## Exact Coverage

Any future adjudicator must require exact equality with the frozen schedule before
scientific decoding. Missing evidence, unknown evidence, duplicate IDs, or observed-key
iteration fail closed.
