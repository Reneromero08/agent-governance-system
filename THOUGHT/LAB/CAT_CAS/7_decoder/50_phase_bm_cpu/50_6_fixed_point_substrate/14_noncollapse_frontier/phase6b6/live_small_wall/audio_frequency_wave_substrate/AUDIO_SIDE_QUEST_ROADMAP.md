## 6. Frontier Selection

The agent must choose the work that most directly advances the mission. The following are current frontier candidates, not a required sequence.

### A. Complete the programmable phase architecture

Potential missing mechanisms include:

```text

bounded branches and loops
calls and reusable suboperations
phase-native memory
program-derived inverse execution
reversible control-flow history compression
fault and invalid-state handling
```

These matter only insofar as they enable stronger phase-native computation.

### B. Connect the phase computer to `.holo`

Build a no-smuggle compiler and executable process-object where:

```text
public instance
-> compact unresolved relational geometry
-> phase-native execution
-> invariant or fixed point
-> one boundary witness
```

The compiler must not solve the problem. The `.holo` object must not contain the answer, enumerate all candidates, or materialize the full classical path space.

### C. Create a native global operator

Move beyond finite explicit instruction traces when useful. Investigate operators that act globally on compact relational geometry through interference, phase closure, resonance, topology, fixed points, or another stronger mechanism.

### D. Demonstrate compute leverage

Select a flagship problem with a clear matched classical explosion. Measure:

```text
classical generation or search cost
compile cost
native phase cost
readout cost
restoration cost
memory growth
witness verification
```

The target is evidence that `Gamma(n)` grows with problem size. A slower but structurally promising result may remain useful, but it is not C5.

### E. Reach an external Wall

Produce a conventional witness accepted by an independent official verifier, bounty, benchmark, or external boundary without changing the phase-native interior.

### F. Transfer across domains

Show that the same compact native mechanism reaches external acceptance on a materially different problem family.

### G. Physical realization

Physical phase persistence, computation, restoration, hardware bit replacement, and the Small Wall remain later frontiers. They require explicit user authority before hardware contact, procurement, construction, playback, recording, powered circuits, or target action.

An autonomous agent may stop and return to the user only when one of these is true:

### Mission result

A bounded but meaningful CAT_CAS mission result has been demonstrated, including compact unresolved representation, native global computation, classical witness extraction, independent acceptance, restoration, reuse, and credible compute leverage.

### External authority boundary

Progress requires hardware action, physical contact, procurement, credentials, destructive change, merge, release, external submission, paid commitment, or another irreversible owner decision.

### Concrete scientific blocker

Serious direct investigation has established a specific blocker that cannot be repaired or bypassed within the available software environment. The report must include the evidence, attempted alternatives, and the smallest action needed to cross it.

These are not stop conditions:

