# Finite Query Equivalence Theorem

Status: `FINITE_QUERY_EQUIVALENCE_ESTABLISHED`

Decision relevance: delayed query selection is necessary custody, but it is not by
itself an identifiability law.

## Objects

Let:

```text
R = relation known to the source before closure
Q = finite public query family {q_0, ..., q_m-1}
h(R, q) = deterministic response law for query q
Y = receiver-measured output
```

The source closes before the receiver selects the realized query `q*`.

## Representation Classes

`answer cache`: a source-authored vector

```text
V(R) = [h(R, q_0), ..., h(R, q_m-1)]
```

The receiver later selects coordinate `V(R)[q*]`.

`relation representation`: a compact state encoding of `R` that does not contain
the query answers as separately addressable coordinates and whose response is
generated by the receiver query operation.

`branch-local additive representation`: separate branch states with response

```text
Y_q(a, b) = F_q(a) + G_q(b) + C_q
```

This has no required pair interaction.

`joint relational representation`: one pair-level state whose response contains a
nonadditive interaction term not reconstructable from branch-local controls:

```text
Y_q(a, b) != Y_q(a, empty) + Y_q(empty, b) - Y_q(empty, empty)
```

`arbitrary lookup table`: an unbounded table indexed by any relation and query.
Finite observations cannot reject this class.

`bounded old-boundary predictor`: an explicitly capacity-limited or complexity-limited
predictor using only the inputs allowed by `B_old`.

## Theorem

For a finite public query family `Q` and a source that knows `R` before closure, the
source can precompute:

```text
V(R) = [h(R, q) for q in Q]
```

If the physical preparation can encode `V(R)`, then a receiver choosing `q` after
source closure can select one coordinate. Therefore delayed query selection alone
does not distinguish a relational state from an answer cache.

## Assumptions

1. `Q` is finite and known or enumerable by the source before closure.
2. `R` is known to the source before closure.
3. The response law `h` is computable by the source before closure.
4. Constructive equivalence requires enough preparation capacity to encode the
   response vector at the precision being scored.
5. The receiver output is scored only on the realized coordinate `q*`.

## Proof

For every `q_i in Q`, the source computes `v_i = h(R, q_i)` before closure. It prepares
a carrier state `P_cache(R)` whose receiver-readable coordinates are `v_i`. The
receiver later draws `q* = q_j` after source closure and reads coordinate `j`.

The observed value is:

```text
Y = read(P_cache(R), q_j) = v_j = h(R, q_j)
```

This is observationally identical to a state that computes `h(R, q_j)` only after the
query, for all observations contained in the closed finite set `Q`. The closure time
of the source therefore does not decide between the two mechanisms when the answer
vector is an admissible preparation.

If a smaller independently measured preparation-capacity bound has not been frozen,
the experiment has not proven that the answer vector is inadmissible. That is an
epistemic non-identification corollary, not a constructive premise of the theorem.

## What The Theorem Does Not Say

It does not say that every ordinary explanation is admissible. It says that the
ordinary answer-cache explanation remains admissible until one of the following is
frozen and passed:

```text
capacity separation:
  minimum ordinary answer-equivalent encoding bits > measured preparation capacity
  relation-representation bits required <= measured preparation capacity

held-out generalization:
  prepared state supports relations, query bases, query parameters, seeds, or
  sessions outside the adversary training cross-product

joint-state intervention:
  branch-local additive controls are null while a pair-level interaction is nonzero

physical mechanism:
  exact Family 10h preparation/query/restoration state is identified and survives
  route, bank, address, order, timing, and orientation controls
```

## Result

The theorem is established. Query separation remains required custody, but it is not
an identifiability proof against a finite answer cache.
