# Unresolved Relational Object

Object:

```text
OrbitState = unresolved fold relation {a, N-a}
```

The object is not a candidate winner. It is a two-branch relation whose branches
remain one process-object until the receiver applies the frozen query at the declared
collapse boundary.

## May Carry

The process-object may carry:

```text
relation basis
both fold branches
composition/path history
carrier coordinates
restoration references
predeclared invariant family
```

## May Not Carry

The process-object may not carry:

```text
hidden winning branch
candidate truth label
orientation label
recovered member
receiver query answer
precomputed q_theta
sender-authored projection result
post-hoc observable selection
```

## Shared Preparation Requirement

Both branches must contribute to one shared physical preparation or evolution. A
design that prepares one carrier for `a` and another carrier for `N-a`, then asks
which scalar is larger, is collapsed before the boundary.

A proposal that merely splits `q_theta` into multiple scalar channels is also
collapsed. It may be a communication or transduction experiment, but it is not a
Small Wall access-model experiment.

## Label Symmetry

Let the relation be represented as:

```text
R = unordered_pair(a, N-a)
```

A label permutation:

```text
candidate_0 <-> candidate_1
```

must leave physical preparation equivalent when the relation itself is unchanged.
If label swapping changes preparation, the design has smuggled orientation or branch
identity.

## Relation Mutation

A true relation-basis mutation changes the relation while preserving labels and
marginal workload. The selected observable must change in the predeclared direction.

This separates relational preparation from:

```text
label dictionary attacks
total source work
bank/core/order lookup
precomputed query answers
```
