# Local Agent Runbook — Phase 6 Executor Engineering Verification

This runbook verifies the combined-observability executor without starting the scientific campaign.

```text
engineering verification ≠ acquisition authorization
host control-state restoration ≠ physical carrier restoration
12 validation-only sessions ≠ 12-session hardware acquisition
```

The full Git repository exists only in the local agent environment. CAT_CAS is a source-transfer and execution target; do not clone the repository or require Git there.

## 1. Local commit proof

On the PR branch, choose the exact head SHA and require a clean tree:

```bash
git branch --show-current
git rev-parse HEAD
git status --porcelain
git cat-file -t "$HEAD_SHA"
git diff --check
```

Stop unless the branch, SHA, empty status, commit object, and diff check are all correct.

## 2. Build the source-transfer bundle locally

From the full local repository:

```bash
python combined_observability_campaign/make_executor_source_bundle.py \
  --repo-root "$REPO_ROOT" \
  --commit "$HEAD_SHA" \
  --plan-dir "$PLAN_DIR" \
  --binding gate_r/COMBINED_CAMPAIGN_BINDING.json \
  --output "$SOURCE_TRANSFER"
```

The builder must reconstruct each transferred source with `git show`, compare the working blob to the committed blob, compile exactly 12 session bundles, reject generated bytecode, and produce:

```text
source_bundle.json
source_bundle.sha256
artifact_class = ENGINEERING_BUNDLE_NOT_ACQUISITION_AUTHORIZATION
acquisition_authorized = false
restoration_authorized = false
```

Independently verify it locally:

```bash
python combined_observability_campaign/make_executor_source_bundle.py \
  --verify-bundle "$SOURCE_TRANSFER" \
  --commit "$HEAD_SHA"
```

Transfer the sealed directory to a new CAT_CAS path. Do not transfer an unbound replacement file afterward.

## 4. Target strict and sanitizer builds

Never compile inside the sealed transfer tree. Copy the runtime sources into new evidence build directories.

```bash
mkdir -p "$TARGET_EVIDENCE/build/strict" "$TARGET_EVIDENCE/build/sanitizer" "$TARGET_EVIDENCE/evidence"
cp "$RUNTIME"/combined_pdn_hardware.h \
   "$RUNTIME"/combined_pdn_hardware.c \
   "$RUNTIME"/combined_pdn_runner.c \
   "$RUNTIME"/test_combined_pdn_runner.py \
   "$RUNTIME"/make_engineering_smoke_schedule.py \
   "$TARGET_EVIDENCE/build/strict/"
cp "$TARGET_EVIDENCE/build/strict/"* "$TARGET_EVIDENCE/build/sanitizer/"
```

Strict:

```bash
(
  cd "$TARGET_EVIDENCE/build/strict"
  gcc --version
  python3 --version
  gcc -std=c11 -O2 -pthread -Wall -Wextra -Werror \
    combined_pdn_runner.c combined_pdn_hardware.c \
    -o combined_pdn_runner -lm
  python3 test_combined_pdn_runner.py
) >"$TARGET_EVIDENCE/evidence/strict_tests.log" 2>&1
```

Sanitizers:

```bash
(
  cd "$TARGET_EVIDENCE/build/sanitizer"
  gcc -std=c11 -O1 -g -pthread -Wall -Wextra -Werror \
    -fsanitize=address,undefined \
    combined_pdn_runner.c combined_pdn_hardware.c \
    -o combined_pdn_runner -lm
  ASAN_OPTIONS=detect_leaks=1 \
  UBSAN_OPTIONS=halt_on_error=1 \
    python3 test_combined_pdn_runner.py
) >"$TARGET_EVIDENCE/evidence/sanitizer_tests.log" 2>&1
```

Copy the final strict binary to the target evidence root and record its digest:

```bash
cp "$TARGET_EVIDENCE/build/strict/combined_pdn_runner" \
   "$TARGET_EVIDENCE/combined_pdn_runner"
sha256sum "$TARGET_EVIDENCE/combined_pdn_runner"
```

Run the transferred Python tests with bytecode disabled and preserve the log:

```bash
(
  cd "$CAMPAIGN"
  PYTHONDONTWRITEBYTECODE=1 python3 -B -m unittest \
    test_campaign_plan.py \
    test_session_schedule.py \
    test_session_determinism.py \
    test_orchestrator.py \

    test_executor_source_bundle.py \

```

## 7. Three-window engineering smoke only

Generate the exact smoke schedule:

```bash

  "$TARGET_EVIDENCE/smoke_schedule"
```

The C executor permits unauthorized real hardware only when `--engineering-smoke` is supplied **and** the schedule exactly matches the frozen two-low-amplitude-plus-one-sender-off shape.

```bash
"$TARGET_EVIDENCE/combined_pdn_runner" \
  --session-dir "$TARGET_EVIDENCE/smoke_schedule" \
  --output-dir "$TARGET_EVIDENCE/smoke_run" \
  --victim 4 \
  --sender 5 \
  --executor-commit "$HEAD_SHA" \
  --pin-khz 1600000 \
  --slot-s 0.5 \
  --off-window-s 0.5 \
  --read-hz 4000 \
  --temp-veto-c 68 \
```

Use the exact source-transfer digest printed by `sha256sum source_bundle.json`:

```bash

python3 -B "$CAMPAIGN/collect_target_engineering_evidence.py" finalize \
  --evidence-root "$TARGET_EVIDENCE" \

  --smoke-run "$TARGET_EVIDENCE/smoke_run" \
  --late-sender-run "$TARGET_EVIDENCE/late_sender_run" \
  --runner "$TARGET_EVIDENCE/combined_pdn_runner" \
  --executor-commit "$HEAD_SHA" \
  --source-transfer-bundle-sha256 "$SOURCE_TRANSFER_SHA" \
  --space-path "$TARGET_EVIDENCE" \
```

This recomputes shared-origin timing, sender lifecycle, true sender-off behavior, capture bounds, cpufreq/boost restoration, process cleanup, and the late-sender result from raw artifacts.
