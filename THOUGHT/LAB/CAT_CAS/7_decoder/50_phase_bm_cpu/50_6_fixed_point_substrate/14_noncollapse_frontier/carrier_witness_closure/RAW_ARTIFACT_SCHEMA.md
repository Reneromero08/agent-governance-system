# Carrier-Witness Raw Artifact Schema

**Schema family:** `CAT_CAS_PDN_CARRIER_WITNESS`  
**Version:** `1.0.0`  
**Binary byte order:** little-endian  
**Binary sample record:** 16 bytes

---

## 1. Campaign directory

```text
<campaign_id>/
  campaign.json
  source_manifest.json
  runs/
    <run_id>/
      run.json
      schedule.json
      windows.csv
      raw_samples.bin
      summary.csv
      analysis.json
      stdout.log
      stderr.log
      run_manifest.json
  aggregate/
    aggregate.json
    closure_report.json
  campaign_manifest.json
```

All JSON files are UTF-8, LF-terminated, with sorted keys where generated by Python. CSV uses UTF-8, comma delimiter, and LF line endings.

---

## 2. Raw binary record

`raw_samples.bin` is a concatenation of fixed-size records:

```c
struct RawSampleLE {
    uint64_t t_tsc_le;
    double   ro_period_le;
};
```

Equivalent Python format:

```python
struct.Struct("<Qd")
```

Record size:

```text
16 bytes
```

`ro_period` is the measured TSC-span divided by the inner ring-oscillator iteration count, matching the original acquisition code.

The binary contains no header. `windows.csv` supplies record offsets and counts. File size must equal:

```text
16 * total_sample_records
```

---

## 3. `windows.csv`

One row per `(symbol, bin)` capture window.

Required columns:

```text
window_index
sample_offset_records
sample_count
symbol_index
bin_index
family
declared_mode
actual_mode
trial
theta_idx
tone_hz
drive_sign
phase_fraction
control
slot_start_tsc
capture_deadline_tsc
first_sample_tsc
last_sample_tsc
temp_before_c
temp_after_c
cur_khz_before
cur_khz_after
cofvid_pstate_before
cofvid_pstate_after
computed_I
computed_Q
computed_magnitude
computed_floor
```

Rules:

- `window_index` starts at zero and is contiguous.
- `sample_offset_records` starts at zero and is contiguous across rows.
- `sample_count >= 4`.
- `family` is one of `preamble`, `real`, `pseudo`, `wrong`, `control`.
- `control` is one of `matrix`, `silent`, `scramble`.
- `declared_mode` and `actual_mode` use the canonical names `basis`, `rotation`, `residual`, `mini`.
- `drive_sign` is `+1`, `-1`, or `0` for silent.
- `phase_fraction` is in `[0,1)`.
- thermal values are finite real Celsius values from the approved sensor.
- frequency values are integer kHz or `-1` only when the field is explicitly unavailable and the campaign contract allowed it before acquisition.
- `computed_*` values are those emitted by the acquisition process and later independently reconstructed.

## 6. `summary.csv`

This preserves compatibility with the original T300 analyzer:

```text
family,declared_mode,actual_mode,trial,hash_restored,theta_idx,
b00_I,b00_Q,...,b11_I,b11_Q,fl00,...,fl11
```

The file must be regenerated from `raw_samples.bin` and `windows.csv`. The validator writes a temporary reconstructed summary and compares it numerically with the retained file.

## 7. Manifests

### `run_manifest.json`

```json
{
  "schema_id": "CAT_CAS_PDN_CARRIER_RUN_MANIFEST_V1",
  "run_id": "...",
  "source_commit": "...",
  "files": {
    "run.json": {"size": 0, "sha256": "..."},
    "schedule.json": {"size": 0, "sha256": "..."},
    "windows.csv": {"size": 0, "sha256": "..."},
    "raw_samples.bin": {"size": 0, "sha256": "..."},
    "summary.csv": {"size": 0, "sha256": "..."},
    "analysis.json": {"size": 0, "sha256": "..."},
    "stdout.log": {"size": 0, "sha256": "..."},
    "stderr.log": {"size": 0, "sha256": "..."}
  }
}
```

The manifest does not hash itself.

### `campaign_manifest.json`

Hashes `campaign.json`, `source_manifest.json`, every run manifest, and aggregate outputs. Large raw files remain transitively bound through their run manifests.

```json
  "primary_route": "v4s5",
  "comparator_routes": ["v2s3"],
  "required_seeds": [0,1,2,3,4,5],
  "required_conditions": ["matrix","silent","scramble"],
  "claim_ceiling": "RECONSTRUCTABLE_CHANNEL_CARRIER_ONLY",

```

The campaign is frozen before the first full run. Any source/binary/configuration change creates a new campaign ID.

## 9. Closure report

`aggregate/closure_report.json` must contain:

```text
structural validity per run
raw reconstruction error maxima
missing run matrix
scientific gate results per seed/route
control behavior
route-scoped closure status
claim ceiling
forbidden claims
campaign-manifest SHA-256
source commit
```

Allowed status values:

```text
PENDING
PARTIAL
CLOSED_ROUTE_4_5
CLOSED_MULTI_ROUTE
INVALID
```
