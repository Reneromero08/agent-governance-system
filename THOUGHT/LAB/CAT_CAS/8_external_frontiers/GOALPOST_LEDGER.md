## Ledger rules

- Prize amounts, records, and deadlines are snapshots, not permanent truths.
- Each active experiment must instantiate `SOURCE_MANIFEST.md` before work begins.
- `OPEN` here means the CAT_CAS experiment is open, not that an external prize is guaranteed open.
- External status must be rechecked from official sources at activation and submission.
- The complete factor set remains visible; no single priority score replaces it.

## Activation factors

For every experiment, record each factor explicitly.

### External clarity

- Is the official object exact?
- Is the verifier or review path known?
- Are eligibility and disclosure conditions clear?

### Minimum falsifiable prototype

- Can the core mechanism be tested on a bounded instance?
- Does failure map a useful blocker?

### Current hardware fit

- Can the first gate run on current CPU/GPU/RAM?
- Is external compute available under the rules?

### Mechanism transfer value

- Does the frontier test target coupling, restoration, unresolved geometry, invariant extraction, or boundary change?

### Deadline pressure

- Is there a closing submission phase or expiring prize?

### Expected cost

- engineering time;
- storage;
- compute;
- expert review;
- publication burden;
- opportunity cost.

## Promotion states

Each experiment moves through:

```text
OPEN
→ SOURCE_FROZEN
→ ADAPTER_VERIFIED
→ PROCESS_OBJECT_LIVE
→ MECHANISM_TESTED
→ SUBMISSION_READY
→ EXTERNALLY_ADJUDICATED
→ TRANSFER_REVIEWED
```

These workflow states do not replace CAT_CAS's canonical experiment status vocabulary. The experiment's canonical status remains one of:

- DONE
- DONE-UNVERIFIED
- COSMETIC
- OPEN
- DEFERRED
- UNKNOWN

## Prize administration ledger fields

When a prize-bearing frontier activates, add:

- official amount;
- payment split;
- current record;
- deadline;
- jurisdiction/eligibility;
- open-source requirement;
- publication requirement;
- compute restrictions;
- IP/disclosure terms;
- contact channel;
- last verified date.

No agent may quote a prize amount from memory as current.

---

## Claim ledger fields

Every frontier report must state:

- official target solved or not;
- official verifier/reviewer response;
- independent verification status;
- evidence level E0–E7;
- transfer level T0–T5;
- exact restoration class;
- known alternative explanations;
- unresolved blockers;
- strongest licensed claim;
- explicitly forbidden stronger claims.
