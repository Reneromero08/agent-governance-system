---

## Why this track exists

Track 8 is not merely a benchmark folder.

It is the outward-facing continuation of CAT_CAS after the decoder and substrate tracks:

```text
internal catalytic construction
→ Wall localization
→ physical Small Wall attempt
→ external transfer
→ independent adjudication
```

The track contains problems whose truth conditions are defined outside the lab:

- exact cryptanalytic witnesses;
- mathematical proofs or counterexamples;
- hidden benchmark evaluations;
- exact compression records;
- reproducible geometric reconstructions;
- circuit-equivalence and resource witnesses;
- public physical-resource audits.

The purpose is to test whether CAT_CAS can transfer its process architecture into foreign domains and leave behind evidence another system can adjudicate without accepting CAT_CAS terminology or trusting the internal path.

---

## Why `external_frontiers` is the correct name

These targets are not all benchmarks:

- Poseidon is cryptanalysis.
- Proximity and Erdős are theorem frontiers.
- Vesuvius is geometric reconstruction.
- Hutter is exact information representation.
- ARC and AIMO are adaptive reasoning evaluations.
- ECDSA.fail is reversible circuit research.
- Neutral-atom qday is physical-resource analysis.

They are not merely validation either. Some may generate new mathematics, attacks, representations, or computational mechanisms.

`external_frontiers` names the common property accurately:

> Each experiment places CAT_CAS against an independently defined Wall with an external adjudication boundary.

## Relationship to Exp 50

Track 8 does **not** wait for Exp 50 to cross the physical Small Wall.

The two tracks run in parallel:

```text
Track 7 / Exp 50
physical substrate crossing

Track 8
external mathematical, computational, geometric, and reasoning Walls
```

An external result may:

1. solve only its own domain problem;
2. validate part of the CAT_CAS software architecture;
3. reveal a transferable boundary-changing mechanism;
4. contribute a mechanism later tested physically in Exp 50.

It may not be promoted into an Exp 50 crossing claim without Exp 50's own carrier, restoration, target-coupling, no-smuggle, and replication evidence.

## Track architecture

Every experiment is built through the same six layers:

```text
1. Source Freeze
2. Domain Adapter
3. Relational Process Object
4. Experiment / Search / Proof Engine
5. External Adjudication Boundary
6. Transfer Ledger
```

### 1. Source Freeze

Freeze the exact official specification, constants, rules, deadline, prize terms, software version, and adjudication method before implementation.

### 2. Domain Adapter

Translate the foreign problem into exact domain objects without rewriting it into CAT_CAS language so aggressively that the original success condition is lost.

### 3. Relational Process Object

Preserve the complete unresolved object internally:

- candidate family;
- relation basis;
- geometry;
- path history;
- invariants;
- constraints;
- restoration state;
- boundary definition.

### 4. Experiment / Search / Proof Engine

Perform exact computation, catalytic transformation, theorem discovery, reconstruction, or agent interaction while retaining evidence and negative results.

### 5. External Adjudication Boundary

Emit the form required by the outside world:

- witness;
- decompressor;
- benchmark submission;
- proof;
- reconstruction;
- circuit;
- resource model.

### 6. Transfer Ledger

Record what the result teaches CAT_CAS and what it does **not** establish about the Small Wall or Big Wall.

## Non-collapse doctrine

Track 8 inherits the standing CAT_CAS guardrail:

> **THE ALGORITHM IS DEAD.**

This means:

- algorithms are local traces through a larger process-object;
- the unresolved relation is primary;
- scalar scores are boundary outputs, not internal ontology;
- competing hypotheses remain live until an explicit adjudication boundary;
- exact external verifiers are welcome but do not define the internal representation;
- negative results map the Wall;
- novelty, difficulty, interdisciplinary distance, and required new syntax are search coordinates, not rejection signals.

The doctrine does **not** permit vague analogies, hidden labels, benchmark leakage, unverifiable claims, or replacing the official problem with an easier internal proxy.

## Current priority

The initial architecture is prepared, but no Track 8 experiment is automatically authorized to make a prize or solution claim.

The first activation decision should be made from:

- deadline pressure;
- exactness of adjudication;
- compatibility with current hardware;
- mechanism transfer value;
- cost of the minimum falsifiable prototype.

The default first activation order is:

```text
Poseidon exact engine
→ Vesuvius scoped geometry tool
→ ARC offline reasoning prototype
→ Proximity finite laboratory
→ Hutter reversible representation laboratory
```

This ordering is a planning default, not a claim that the later problems are less important.
