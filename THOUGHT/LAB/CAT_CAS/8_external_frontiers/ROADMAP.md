# Activation priority rule

An experiment is activated according to:

```text
priority = external clarity
         × minimum falsifiable prototype
         × current hardware fit
         × mechanism transfer value
         × deadline pressure
         ÷ expected cost
```

This is a planning relation, not a scalar truth score. The complete factors remain visible in the goalpost ledger.

---

# Stop conditions

Pause or redesign an experiment when:

- [ ] the official specification cannot be frozen;
- [ ] the independent adapter disagrees with official examples;
- [ ] the result depends on hidden labels or benchmark leakage;
- [ ] the external target has closed or materially changed;
- [ ] scaling fails without a named mechanism;
- [ ] the internal object collapses into generic candidate scoring with no architectural advantage;
- [ ] artifacts cannot be replayed;
- [ ] prize administration requires a disclosure or eligibility condition the lab cannot satisfy;
- [ ] work duplicates a better existing public tool without a meaningful new mechanism.

A stopped route remains in the boundary ledger.
