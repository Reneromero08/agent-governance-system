# Exp 57 — Erdős Bounties

**Status:** OPEN  
**Adjudication:** Class D proof or counterexample  
**Role:** theorem-discovery ladder between exact finite laboratories and grand foundational problems

---

# Frontier object

This experiment is not one theorem. It is a governed selector for finite, combinatorial, number-theoretic, graph-theoretic, and extremal problems carrying explicit bounties.

The process-object is the full extremal family and its transformations, not only the current best witness.

Proposed objects:

- `ProblemIntakeObject`
- `ExtremalOrbit`
- `WitnessFamily`
- `BoundaryAtlas`
- `InvariantFamily`
- `LemmaGraph`

---

# Selection criteria

Prefer problems with:

- exact finite formulation;
- accessible definitions;
- known upper/lower bounds with a gap;
- computationally generable examples;
- tractable symmetry quotient;
- active bounty with identifiable administrator;
- potential for proof or counterexample;
- no immediate dependence on an enormous unbuilt theory.

Priority domains:

- graph theory;
- additive combinatorics;
- extremal set systems;
- number-theoretic sequences;
- Ramsey-type structures;
- discrepancy;
- finite geometry;
- polynomial incidence.

# Activation gates

## Gate 0 — Prize/source freeze

- [ ] current database entry archived;
- [ ] exact statement and variants frozen;
- [ ] bounty amount and administrator confirmed;
- [ ] current solved/unsolved status confirmed;
- [ ] literature cutoff recorded;
- [ ] specification digest created.

## Gate 1 — Candidate scoring

Score at least twenty problems across:

- [ ] definition accessibility;
- [ ] exact finite foothold;
- [ ] current-bound gap;
- [ ] symmetry leverage;
- [ ] compute compatibility;
- [ ] theorem-extraction potential;
- [ ] bounty clarity;
- [ ] domain transfer value.

Select three, not one.

## Gate 2 — Exact finite laboratory

For each selected problem:

- [ ] generator;
- [ ] exact predicate verifier;
- [ ] canonical labeling or symmetry reduction;
- [ ] exhaustive small parameter sweep;
- [ ] witness database;
- [ ] known result reproduction;
- [ ] negative-result preservation.

## Gate 3 — Boundary atlas

Map:

- [ ] extremal values;
- [ ] transition parameters;
- [ ] minimal obstructions;
- [ ] orbit stabilizers;
- [ ] near-extremal families;
- [ ] failed candidate lemmas;
- [ ] recursive constructions.

## Gate 4 — Invariant discovery

Search:

- [ ] spectral invariants;
- [ ] rank;
- [ ] modular structure;
- [ ] entropy inequalities;
- [ ] forbidden substructures;
- [ ] topological summaries;
- [ ] polynomial identities;
- [ ] additive energy;
- [ ] matroid structure.

## Gate 5 — Generalization

- [ ] state candidate lemma;
- [ ] adversarially test finite cases;
- [ ] derive induction, stability, or recursive construction;
- [ ] separate computational evidence from proof;
- [ ] obtain independent mathematical review;
- [ ] formalize critical finite or symbolic steps where practical.

## Gate 6 — Bounty adjudication

- [ ] contact administrator;
- [ ] submit exact proof/counterexample;
- [ ] satisfy publication/review terms;
- [ ] preserve administrative and scientific ledgers separately.

---

# Fastest falsifiable prototypes

- disprove a naive strengthening with an exact counterexample;
- improve one finite bound;
- identify a new extremal family;
- reproduce and extend a known computational table;
- prove a restricted parameter case;
- find a recursive construction matching current best values.

---

# No-smuggle model

Forbidden:

- using a known solution without provenance;
- treating an unverified database status as current;
- finite evidence promoted to universal proof;
- hidden exhaustive cutoffs;
- selecting only parameters supporting the conjecture;
- omitting counterexamples to intermediate lemmas;
- changing the problem variant after results appear.
