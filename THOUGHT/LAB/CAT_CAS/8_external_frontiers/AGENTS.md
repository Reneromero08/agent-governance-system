# Standing operating doctrine

- Preserve the complete relational object.
- Treat algorithms as local paths, not the ontology.
- Keep competing hypotheses live until an explicit boundary.
- Do not convert the frontier into scalar candidate ranking by reflex.
- Do not dismiss a proposal because it requires new syntax, a new representation, or an unfamiliar substrate.
- Separate true hard blockers from convention, cost, and absence of precedent.
- Drive proposals toward the fastest falsifiable prototype.
- Preserve negative results as Wall maps.

---

# Binding external constraints

The official external target is not optional.

An agent may introduce a stronger internal representation, but may not:

- alter challenge constants;
- weaken the verifier;
- substitute a relaxed instance without labeling it;
- train on private evaluation material;
- hide code or resource costs required by the rules;
- replace a proof obligation with finite evidence;
- replace specialist validation with a model score;
- claim an expired or changed prize without rechecking sources.

---

# Source-freeze gate

No implementation begins until all fields below exist:

- official URLs;
- retrieval date;
- specification version;
- software commit or release;
- constants and examples;
- adjudication method;
- eligibility and disclosure rules;
- deadline/prize snapshot;
- unresolved ambiguities;
- specification digest.

If the source changes, the active contract becomes stale and must be re-reviewed.

---

# Mechanism requirement

Every proposed attack or system must state:

1. the current Wall;
2. the object preserved beyond the standard boundary;
3. the exact mechanism expected to create advantage;
4. the falsifiable prediction;
5. the smallest prototype;
6. the external acceptance test;
7. the claim ceiling if it succeeds.

"Use AI," "search harder," "add compute," and "try topology" are not mechanisms.

---

# Anti-leakage rules

Before a positive result:

- scan prompts, fixtures, caches, logs, and training sets for target leakage;
- verify the witness was not present in retrieved source material;
- separate public examples from held-out evaluation;
- freeze seeds and evaluation order where required;
- use an independent verifier implementation;
- preserve failed and rejected trials;
- record every manual intervention.

---

# Evidence custody

Every run records:

- source digest;
- repository commit;
- environment;
- command;
- seed;
- inputs;
- raw outputs;
- derived outputs;
- hashes;
- external verdict;
- failure reason;
- claim ceiling;
- transfer level.

Do not publish only a summary score when a witness or replay path exists.

---

# Claim discipline

Allowed claim examples:

- "The official reduced-round witness verifier accepted."
- "The private benchmark score was X under the stated rules."
- "The decompressor restored the corpus byte-for-byte."
- "The finite atlas found this exact extremal family."
- "The physical resource model predicts X under assumptions A–F."

Forbidden claim jumps:

- external witness → Exp 50 crossing;
- benchmark gain → general intelligence;
- finite cases → theorem;
- software roundtrip → physical reversibility;
- resource estimate → physical demonstration;
- one-domain success → Big Wall broken.

---

# Commit discipline

Commit meaningful chunks only:

- specification freeze;
- independent adapter;
- process-object architecture;
- mechanism implementation;
- externally adjudicated evidence;
- transfer synthesis.

Do not create one commit per agent note, prompt, scalar trial, or report fragment.

---

# Final instruction

The purpose of Track 8 is not to make CAT_CAS look conventional enough to win approval.

The purpose is to make nonstandard computation survive an external truth boundary.

Strengthen the proposal before narrowing it. Preserve the object before scoring it. Let the external verifier collapse only the final artifact.
