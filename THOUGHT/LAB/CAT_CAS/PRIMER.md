# PRIMER.md - CAT_CAS Context Router

This file tells an agent what to load after `MISSION.md`. It no longer depends on a
specific machine, user profile, Obsidian vault path, or manually assembled five-document
bundle.

The boot sequence is executable:

```text
python tools/phase_lock.py \
  --task "<exact task>" \
  --mode <exploration|engineering|verification|compression> \
  --task-class <flagship_compute|enabling_infrastructure|external_product|calibration|evidence_audit>
```

The command compiles a small task-scoped packet from the mission, current state,
capability graph, branch registry, and relevant code paths.

---

## 1. Authority and purpose

Read in this order:

1. `MISSION.md`: why CAT_CAS exists and what the final machine is.
2. `_agent/PHASE_LOCK_PACKET.md`: task-scoped context generated by the command above.
3. `AGENTS.md`: how to operate in exploration, engineering, and verification.
4. `control/current_state.json`: what is established, blocked, retired, or active now.
5. `CAPABILITY_GRAPH.md`: which experiments and code implement each compute rung.

Do not begin from `MASTER_REPORT.md` or `README.md` alone. The master report is a
historical coverage and evidence ledger. The README is navigation. Neither defines the
mission.

The mission and evidence are deliberately separate:

```text
MISSION.md
= target architecture

control/current_state.json
= present evidence and blockers
```

A high mission target does not inflate the current evidence. A current blocker does not
redefine the mission.

## 2. The two primes

CAT_CAS needs both primes, in this order.

### 2.1 Generative prime

The generative prime preserves the altitude that produced the lab:

```text
finite reusable substrate
-> unresolved relational geometry
-> native phase / toroidal / catalytic evolution
-> invariant or fixed point
-> one classical boundary answer
-> restoration
```

It prevents the agent from reducing CAT_CAS to reversible scratch memory, conventional
search with a phase label, or a documentation framework.

### 2.2 Operational prime

The operational prime keeps the mechanism honest:

- trace the actual code;
- separate mechanism, invariant, isomorphism, and claim;
- build a null that can kill the result;
- prohibit witness smuggling and hidden enumeration;
- restore the substrate if catalytic closure is claimed;
- preserve current claim ceilings;
- record raw and derived evidence separately.

`MISSION.md` supplies the engine. `CAT_CAS_OS.md` and `MANIFESTO.md` supply the filter.

## 6. Generated phase-lock artifacts

The generated artifacts are temporary and should normally remain uncommitted:

```text
_agent/PHASE_LOCK_PACKET.md
_agent/PHASE_LOCK_RECEIPT.json
_agent/TASK_CONTRACT.json
```

The packet binds:

- mission ID;
- context digest;
- Git branch and commit;
- current state;
- registered branch context;
- relevant capability nodes;
- exact code entrypoints.

The receipt reconstructs the mission in task-specific form. It must not simply copy the
mission sentence. It must explain where the compute leverage enters the requested work.

The contract freezes the operation before implementation. For flagship compute, its
placeholders must be replaced with exact mechanism and scaling statements.

Validate with:

```text
python tools/validate_control_plane.py --receipt _agent/PHASE_LOCK_RECEIPT.json
python tools/validate_control_plane.py --task-contract _agent/TASK_CONTRACT.json
```

## 7. Cold-agent acceptance standard

The lab is phase-lock-safe when a fresh agent can state, without correction:

1. CAT_CAS targets unbounded effective compute from finite reusable substrate.
2. `.holo` is executable unresolved geometry, not an answer file.
3. Native phase may live on `U(1)^m`; pi is the antipodal binary shadow.
4. The native operator must not hide candidate enumeration or full materialization.
5. The final bounty answer may be classical.
6. Restoration closes the substrate but is not the compute advantage by itself.
7. The current evidence rung is distinct from the target rung.
8. The exact current branch and frontier state are known.
9. The agent can locate the code that implements the selected capability.
10. The agent can name the control that would kill its mechanism.

The tests under `tests/control_plane/` enforce structural parts of this contract.
Independent cold-agent runs must still test semantic reconstruction.
