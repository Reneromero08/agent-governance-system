--- VERDICT ---
If the bigint addition cost jumps at 256->300 (9->10 limbs), the spike is from
Python's internal bigint digit array crossing an allocator boundary, NOT from
CPU cache lines (64 bytes = 512 bits) or memory pages (4096 bytes).

Cache line = 64 bytes = 512 bits. If the spike were from cache, it would be at 512.
Python limb = 30 bits. 256 bits = 9 limbs. 300 bits = 10 limbs.
If the spike is at 300 bits, the mechanism is Python bigint, not CPU hardware.
