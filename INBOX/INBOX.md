# Process

1. **Inbox Entry:** All incoming docs start here.
2. **Metadata:** Every entry MUST have a list item in the sections above.
3. **Hash:** Every document MUST start with `<!-- CONTENT_HASH: ... -->` (Line 1).
4. **Approval:** Governance checks hash, User approves content.
5. **Promotion:** Move to permanent location (CANON, CONTEXT, etc).

## Authority Rule

No document bypasses INBOX. Hash integrity is enforced by pre-commit hooks.
